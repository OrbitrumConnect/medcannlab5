## 🧠 VOLUME III: O CÓRTEX TÉCNICO (ARQUITETURA COGNITIVA)

### 3.1. TradeVision Core (O Governador)
Localizado em `supabase/functions/tradevision-core`, o sistema possui 14 subsistemas:
- **Intent Decoder:** Traduz o tom de voz do paciente para intents de sistema.
- **Normalization Layer:** Limpa os dados de entrada antes do processamento.
- **ML Risk Prediction:** Detecta sinais de alerta (suicídio, dor crítica) via `risk_cockpit`.
- **Trigger Engine:** Dispara widgets visuais (`SchedulingWidget`, `PaymentCard`) via tags GPT: `[TRIGGER_SCHEDULING]`.

### 3.2. COS — Cognitive Operating System
O "Sistema Operacional da Consciência" do MedCannLab:
- **Auditabilidade Total:** Cada decisão da IA é imutável em `cognitive_events`.
- **Contenção RAG:** A IA só pode falar o que está nos 448 documentos da base de conhecimento.
- **The Governor:** A Edge Function atua como juiz final, filtrando alucinações e negando prescrições automáticas sem médico.

## 🛡️ VOLUME V: O SENTINELA DE DADOS (RLS & BANCO DE DADOS)

### 5.1. O Censo das 136 Tabelas
O MedCannLab opera sobre 136 tabelas estruturadas, divididas em:
- **Subsistema Clínico (35+):** `appointments`, `clinical_assessments`, `clinical_reports`, `cfm_prescriptions`.
- **Subsistema Cognitivo (25+):** `cognitive_events`, `ai_chat_interactions`, `risk_cockpit`.
- **Subsistema Educacional (20+):** `courses`, `lessons`, `user_statistics`.
- **Subsistema Financeiro (30+):** `user_subscriptions`, `transactions`, `billing_profiles`.

### 5.2. Row Level Security (A Muralha)
- **100% de Cobertura:** Nenhuma query atinge o banco sem passar pelo filtro `auth.uid()`.
- **Bypass Admin:** Admins possuem políticas especiais de `BYPASS` para auditoria global.
- **Soft Delete:** Implementado via trigger SQL que impede o `DELETE` físico, convertendo-o em `is_deleted = true`.
