### 4. PDF ICP-Brasil REAL com `/Sig` embedded (V1.9.299 — 15/05)

**Descoberta crítica 15/05**: até hoje os 25+ documentos assinados via edge `digital-signature` (V1.9.176) tinham PKCS#7 válido **sobre JSON** (auditoria interna) mas o PDF que o paciente baixa **não tinha `/Sig` dictionary embedded** conforme ISO 32000-1 §12.8. Validadores externos (validar.iti.gov.br, Adobe Acrobat) rejeitavam como "documento sem assinatura".

**Solução V1.9.299**:
- Edge nova `sign-pdf-icp` paralela (zero regressão)
- Stack manual: `pdf-lib` + `node-forge` + injeção `/Sig` byte-level
- Cadeia ICP-Brasil embedded (AC DigitalSign RFB G3 + AC SRF v4) — fail-safe pra `.pfx` sem chain
- PBAD AD-RB compliance: `signing-certificate-v2` attribute (OID 1.2.840.113549.1.9.16.2.47, RFC 5035) — obrigatório pra ITI reconhecer como ICP-Brasil

**Princípio cristalizado**: *"O cert do Ricardo era real desde sempre; faltava o PDF saber disso."*
