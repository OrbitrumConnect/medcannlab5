## 🔄 Fluxos Principais

### Fluxo 1: Profissional Adiciona Paciente

```mermaid
sequenceDiagram
    participant P as Profissional
    participant APP as NewPatientForm
    participant DB as public.users
    participant CA as clinical_assessments
    participant UR as user_roles

    P->>APP: Preenche dados + email
    APP->>DB: INSERT (type='patient')
    APP->>CA: INSERT (doctor_id=P.id)
    Note over DB: Paciente visível para P via getAllPatients()
    Note over DB: Paciente visível para Admin via admin_get_users_status()
```

### Fluxo 2: Paciente Self-Signup

```mermaid
sequenceDiagram
    participant PAC as Paciente
    participant AUTH as auth.users
    participant TRIGGER as on_auth_user_created
    participant DB as public.users
    participant UR as user_roles

    PAC->>AUTH: Signup (email + senha + type)
    AUTH->>TRIGGER: AFTER INSERT
    TRIGGER->>DB: INSERT (sync name, email, type)
    TRIGGER->>UR: INSERT role (paciente/profissional)
    Note over DB: ✅ Agora visível em TODO o sistema
```

### Fluxo 3: Video Call (WiseCare + Fallback)

```mermaid
sequenceDiagram
    participant U as Usuário
    participant VC as VideoCall.tsx
    participant EF as wisecare-session
    participant WC as WiseCare API
    participant P2P as WebRTC P2P

    U->>VC: Aceita chamada
    VC->>EF: POST /wisecare-session
    EF->>WC: POST /api/auth/org/ (login)
    WC-->>EF: Bearer token
    EF->>WC: POST /api/v1/rooms (criar sala)
    alt WiseCare OK
        WC-->>EF: joinUrl
        EF-->>VC: sessionId + joinUrl
        VC->>VC: Render iframe 🟢
    else WiseCare Falha
        VC->>P2P: Fallback automático 🟡
        P2P-->>VC: WebRTC P2P direto
    end
```
