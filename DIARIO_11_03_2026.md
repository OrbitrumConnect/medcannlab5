## ⚛️ PARTE 2: COS ENGINE — O SISTEMA IMUNOLÓGICO DA NÔA

### `cos_engine.ts` (112 linhas) + `cos_kernel.ts` (14 linhas — re-export)

O COS (Cognitive Operating System) é a camada de governança que decide **"A Nôa pode pensar agora?"** antes de qualquer chamada GPT.

```
FLUXO COS:
                                                 ┌──────────────┐
 Request ──→ system_config (ai_mode) ──→        │  COS_Context  │
             institutional_trauma_log ──→        │  evalute()    │
             cognitive_metabolism ──→             └──────┬───────┘
             cognitive_policies ──→                      │
                                                         ▼
                                               ┌─────────────────┐
                                               │  COS_Decision    │
                                               │  .allowed        │
                                               │  .mode           │
                                               │  .flags          │
                                               │  .autonomy_level │
                                               └─────────────────┘

6 camadas de decisão (em ordem):
1. KILL SWITCH (OFF) → bloqueia tudo
2. TRAUMA LOCK → bloqueia + razão do trauma
3. METABOLIC THROTTLE → limite diário atingido → SILENCE_MODE
4. READ_ONLY → permite leitura, bloqueia escrita
5. POLICY ENFORCEMENT → ação proibida por política
6. AUTORIZADO → córtex liberado
```

### Tabelas envolvidas no COS:

| Tabela | Função |
|:-------|:-------|
| `system_config` | Kill Switch global (ai_mode: FULL/READ_ONLY/OFF) |
| `institutional_trauma_log` | Registro de traumas (ex: falha OpenAI) |
| `cognitive_metabolism` | Limite diário de decisões por profissional |
| `cognitive_policies` | Políticas por intent (autonomy_level, forbidden_actions) |
| `cognitive_decisions` | Átomos de Decisão (PRE-AI → POST-AI) |
| `cognitive_events` | Log de todos os eventos (CEP — append-only) |
| `cognitive_interaction_state` | Estado CAS por usuário (depth_level, traits) |

## 🩺 PARTE 7: A FILOSOFIA CLÍNICA — DR. RICARDO VALENÇA

### O Problema Fundamental da Medicina Atual

Durante a sessão de trabalho, Dr. Ricardo trouxe uma reflexão profunda sobre **por que** o AEC 001 foi desenhado como foi — e por que a Nôa **não pode diagnosticar**.

#### O Modelo Dominante (que Ricardo critica):
```
Sintoma → Hipótese rápida → Diagnóstico → Tratamento
                     ↑
              (15 minutos)
```

#### O Exemplo que Ricardo Deu:
> *"O cara entrou com dor de cabeça, mediu a pressão, estava 18 por 10 — 'ah, isso é dor de cabeça da pressão alta'. Mas podia ser que a mulher dele caiu da cama, quebrou a bacia, ele ficou sem poder ajudar, teve que cuidar dela, da casa e do trabalho. Acordou com dor de cabeça porque preocupado. E o médico nunca vai saber porque nunca perguntou."*

#### O Conceito: Fechamento Diagnóstico Prematuro

Na medicina, isso se chama **premature diagnostic closure** — quando o médico conclui cedo demais e:

| Consequência | Impacto |
|:------------|:--------|
| Ignora outros dados | Perda de contexto |
| Filtra a narrativa | Só ouve o que confirma a hipótese |
| Interrompe o paciente | Mata o relato espontâneo |
| Cria efeito túnel | Viés de confirmação |

> *"O diagnóstico impede a individualização terapêutica. Ele é impeditivo. O diagnóstico impede a pessoa de se manifestar."* — Dr. Ricardo Valença

### A Pergunta Fundamental

> **"Como é que as IAs estimulam o relato espontâneo?"** — Dr. Ricardo Valença

Esta pergunta é, na essência, uma **pergunta de pesquisa médica real**.

### A Resposta: Como a Nôa Faz Diferente

A Nôa foi desenhada com **3 princípios anti-fechamento prematuro**:

| Princípio | Implementação no AEC 001 | Linha no Core |
|:----------|:------------------------|:-------------:|
| **1. Pergunta aberta** | "O que trouxe você aqui?" (não "Você tem dor?") | L1707 |
| **2. Repetição expansiva** | "O que mais?" (repete até o paciente encerrar) | L1707 |
| **3. Ausência de interpretação** | Resumo puramente descritivo, sem rótulos | L1725 |

### O Fluxo Anti-Diagnóstico Precoce

```
Paciente: "Tenho dor de cabeça"

❌ IA CONVENCIONAL:           ✅ NÔA (AEC 001):
─────────────────           ────────────────
"Há quanto tempo?"          "O que mais?"           ← expande
"Que tipo de dor?"          "O que mais?"           ← expande
                            "O que mais?"           ← expande
→ Diagnóstico: „cefaleia    → Paciente: "Minha mulher
  tensional"                  caiu, quebrou a bacia,
                              estou cuidando de tudo
→ 15 minutos                  sozinho..."
→ Caso encerrado            
                            → Queixa real: sobrecarga
                            → Contexto completo
                            → Médico recebe história RICA
```

### Por Que a IA é Perfeita Para Isso

| Vantagem da IA | Limitação Humana |
|:--------------|:-----------------|
| Pergunta sem pressa | Médico tem tempo curto |
| Não interrompe | Humano interrompe por viés |
| Mantém estrutura | Humano desvia por pressão do sistema |
| Repete "O que mais?" sem cansaço | Humano se cansa |
| Não tem viés de confirmação | Humano tem hipótese precoce |

### A IA Como Amplificadora da Escuta

Ricardo define a Nôa **não** como substituta do médico, mas como:

> **"Amplificadora da escuta clínica"** — o paciente chega ao médico com uma história já organizada, rica, completa. O médico parte de um campo narrativo muito mais robusto.

Isso muda a equação:

```
SEM NÔA:                        COM NÔA:
──────────                       ─────────
Paciente → Médico (15min)        Paciente → Nôa (20min) → Médico (15min)
                                          ↑
                                    História completa
                                    Queixas expandidas
                                    Contexto familiar
                                    Hábitos mapeados
                                    Zero diagnóstico
```

### Implicações Para Além do App

O que o MedCannLab está construindo tem implicações para:

| Área | Impacto |
|:-----|:--------|
| **Ensino médico** | Framework de anamnese assistida por IA |
| **Telemedicina** | Escuta pré-consulta estruturada |
| **Triagem clínica** | Narrativa antes de classificação |
| **Pesquisa clínica** | Dados narrativos ricos para estudos |
| **Filosofia médica** | Evidência de que IA pode estimular relato espontâneo |

### Princípio Selado por Ricardo

> *"Só tem um jeito de saber o que está por trás: estimular o relato espontâneo. A doença NÃO é o centro. O centro é a escuta e a narrativa."*

O GPT que gerou os documentos **não leu o `tradevision-core/index.ts`**. Ele partiu do conceito correto mas não conhecia a implementação real. Por isso:

- Propôs 3 modos quando existem 6+

- Falou em "vetorização ativa" quando o core não usa embeddings
- Omitiu completamente: COS, gamificação, predição ML, motor determinístico, CAS
